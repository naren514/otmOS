Definitions - CONTENT_MANAGEMENT_SYSTEM

Content Management System for document integration

CONTENT_MANAGEMENT_SYSTEM_GID
Content Management System GID
CONTENT_MANAGEMENT_SYSTEM_XID
Content Management System XID
DESCRIPTION
Description of system
URL
URL for system communication
TYPE
Type of CMS: "OTM" = OTM is managing and storing documents within the schema. "WCC" - Oracle Webcenter Content is used to manage and store documents. "JCR" - Some other JSR-170 compliant system is used to manage and store documents.
USER_NAME
User name for remote authorization
PASSWORD
Credentials for remote authorization
WCC_SUPPORTS_ACCOUNTS
If "Y", WCC has been configured to support accounts.
WCC_USE_DIRECT_URL
If "Y", users receive direct URL's into WCC, necessitating an SSO solution or login to WCC. If "N", content is piped through OTM.
WCC_USE_AUTO_GENERATED_ID
If "Y", WCC has been configured to auto generated document ID's.
WCC_SECURITY_GROUP
The default security group for documents.
WCC_DOCNAME_GEN_GID
The default generator for the dDocName WCC parameter.
WCC_DOCTITLE_GEN_GID
The default generator for the dDocAccount WCC parameter.
WCC_ACCOUNT_GEN_GID
The default generator for the dDocAccount WCC parameter.
WCC_REVLABEL_GEN_GID
The default generator for the dRevLabel WCC parameter.
WCC_REVIEW_MODE
If "NONE", no document review workflows are used with the CMS. If "WCC", OTM forwards reviewers to WCC which is responsible for managing the review process. Users can accept or deny documents from OTM, but the action is forwarded to WCC. If "OTM", standard OTM review workflow is used. Documents cannot be accept or denied from WCC.
WCC_DISTRIBUTION_MODE
If "NONE", no document distribution is used with the CMS. If "WCC", OTM forwards distribution parties to WCC which is responsible for forwarding them finalized content. Users can finalize or unfinalize documents from OTM, but the action is forwarded to WCC. If "OTM", standard OTM finalization workflow is used. Documents cannot be finalized from WCC.
JCR_CONNECTION_TYPE
Type of connection to retrieve a JSR-170 Repository. "JNDI" assumes the content server supports JNDI lookup. "SERVICE LOADER" assumes libraries to support the content server are included with OTM. "JAVA" assumes a proprietary connection is needed.
JCR_CONNECTION_PLUGIN_GID
If the JCR connection type is "JAVA", the plugin to retrieve a JCR Repository object.
JCR_SUPPORTS_REVISIONS
If "Y", the JSR-170 compliant CMS supports document versioning
ROOT_FOLDER
Root folder for all OTM documents. If null, documents are assumed unfiled.
FOLDER_TYPE
Type of folders for document assignment. If NONE, documents are assumed unfiled. If FIXED, all documents are placed in the root folder. If DOMAIN, documents are placed in a subfolder of the root, equal to the domain. E.g., a document in domain A/B ,with a root of R, is placed in R/A/B. If GENERATED, the subfolder is determined by a parameter generator.
FOLDER_GEN_GID
The parameter generator for document subfolders.
CMIS_BINDING
CMIS Binding. If WEB SERVICES, Web Service calls over SOAP are used to invoke CMIS. If ATOM PUB, RESTful calls are used.
CMIS_USE_VERSIONING
If Y, the CMIS versioning implementation should be used when adding OTM revisions. Otherwise, an independent document is created for each revision.
CMIS_NAME_GEN_GID
Generator for the cmis:name parameter.
CMIS_REVLABEL_GEN_GID
Generator for the cmis:versionLabel parameter.
CMIS_REPOSITORY_ID
CMIS repository ID. If left blank, OTM will use the first repository found.
FOLDER_CMS_TYPE
Custom folder CMS type.
DOCUMENT_CMS_TYPE
Custom document CMS type.
ODC_DOCTITLE_GEN_GID
This column would store the param generator gid to generate title for documents
