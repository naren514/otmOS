Definitions - EXTERNAL_SYSTEM

EXTERNAL_SYSTEM_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
EXTERNAL_SYSTEM_XID
GC3 Unique identifier for the entity. Is used for creating the GID
EXPECTED_RESPONSE_TIME
Used in notify as guaranteed delivery. Set the expected response time in second from receiving external system.
IS_GUARANTEED_DELIVERY
Flag that indicates whether the notification is a guaranteed delivery. A guaranteed delivery is a special type of notification where the receiver sends response back to the sender indicating whether the received message can be /or has been processed.
SOCKET_TIMEOUT
HTTP read/write timeount in seconds. A value of 0 means no timeout.
ACTIVE_MODE
Used to enable or disable sending to the external system. When set to Enabled it will generate and send the xml (default mode); Disable Send will generate and store the xml without sending; Disable Build will log an entry in the i_transmission and i_transaction tables without generating the xml blob; and Disabled will not log, generate, or send.
IS_ENABLE_DEBUG
Specifies if debugging is enabled for the external system.
IS_LOG_RESPONSE_MSG
Specifies if the response received should be logged.
REATTEMPT_MODE
Specifies the type of logic used to setup reattempts to send to the external system. If None, no reattempts are made. If Redo, then the status of the transmission will be set to REDO when an error occurs and the REDO transmission processing will manage a resend. If Delay, then the reattempt_delay field is used to indicate how long to wait before the next attempt.
REATTEMPT_MAX
Specifies the max number of attempts to send the transmission to the external system.
REATTEMPT_DELAY
Specifies the amount of time to wait before another attempt to send can be made.
REATTEMPT_DEADLINE
Specifies the max time from when the transmission is created that an attempt will be made to send the transmission.
MAX_THREADS
Specifies the max number of threads that should be used for this external system. This should help limit a bad destination taking up all the threads.
FTP_APPEND_FILE
Specifies if the FTP protocol should use the append file mode to send the data to the FTP server in batches, or should use the put mode where the whole transmission is sent at once.
HTTP_ACK_EXPECTED
Specifies if the Transmission Ack is expected to be received when sending via synchronous protocol.
HTTP_USE_ACK_STATUS
Specifies if the Transmission Ack status should be used as the status of the Transmission
HTTP_READ_RESPONSE
Specifies if the HTTP response conde should be validated.
SERVICE_ACK_EXPECTED
Specifies if the Transmission Ack is expected to be received when sending via synchronous protocol.
SERVICE_USE_ACK_STATUS
Specifies if the Transmission Ack status should be used as the status of the Transmission
BPEL_ACK_EXPECTED
Specifies if the Transmission Ack is expected to be received when sending via synchronous protocol.
BPEL_USE_ACK_STATUS
Specifies if the Transmission Ack status should be used as the status of the Transmission
BPEL_DOMAIN_PW
Specifies the password for the BPEL domain.
BPEL_JNS_USER
Specifies the java.naming.security.principal property for connecting to the BPEL instance using RMI.
BPEL_JNS_PW
Specifies the java.naming.security.credentials property for connecting to the BPEL instance using RMI.
BPEL_MODE
Specifies if the BPEL notify method should use RMI or SERVICE to send to the BPEL server.
HTTP_CLOSE_CONNECTION
If it is "Y", close the HTTPPOST connection
DELAYED_STREAM_TRANSPORT
If it is "Y", the XML is loaded before sending out and transported through data stream
USE_PROXY_SERVER
Determines whether the messages or request should be passed through the proxy server or not.
INCLUDE_NAMESPACE
NULL, NONE, DEFAULT and STRICT to be allowable values
FTP_PORT
Port # when external system is used for FTP or FTPS
TARGET_NAMESPACE
Indicates which version of the Transmission should be sent to this external system
