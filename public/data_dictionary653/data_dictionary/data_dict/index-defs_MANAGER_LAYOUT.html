Definitions - MANAGER_LAYOUT

This defines a custom screen that is used to view or edit entitys in the system

MANAGER_LAYOUT_NAME
The name of the manager layout record as it pertains to a user
ENTITY_NAME
This is the top level business object that defines the map of child records
POST_SAVE_NAV
This is the location the screen will navigate to after a record has been saved
MANAGER_LAYOUT_XML
The XML that the layout manager UI manipulates to generate child records
MANAGER_LAYOUT_TYPE
The type of Manager Layout, either (E)dit or (V)iew.
IS_TEMPLATE
If the manager layout is a template then the manager_layout_xml will include links to the public files to build the manager layout document. Otherwise the manager_layout_xml will be the marked up version of the layout document
DISPLAY_UOM_AS_ENTERED
If this is true all UOMs in this manager layout will display their values as they were entered instead of converting to user preference
PARENT_MANAGER_LAYOUT_GID
The parent manager layout gid is the template that this record was created from
SUPPORTS_ADD_FIELD
Determines if this manager layout allows fields to be added in the toolkit
TITLE
Title of Page
