Definitions - OB_ORDER_BASE

This is the base table that defines the orders. An order base can have multiple origins, destinations, different times etc. Multiple order releases can be created against one order base. In the XML side, the TransOrder(Transportation Order) is similar to Order base.

ORDER_BASE_GID
ORDER_BASE_GID contains the unique identifier for the order base.
ORDER_BASE_XID
ORDER_BASE_XID contains the unique external identifier for the order base.
EXTERNAL_SYSTEM_ID
The identifier for the external system that provided the base order. For example a users SAP or JDEdward system could be providing the order ids.
PAYMENT_METHOD_CODE_GID
PAYMENT_METHOD_CODE_GID contains the unique identifier for the payment method code for the order. Examples of payment method include electronic fund transfer and cash on delivery.
INCO_TERM_GID
INCO_TERM_GID contains the unique identifier for the incoterms associated with the order base.
TERM_LOCATION_TEXT
TERM_LOCATION_TEXT contains additional information related to the incoterms for the order.
PLANNING_GROUP_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
FIXED_ITINERARY_GID
FIXED_ITINERARY_GID contains the unique identifier for the fixed itinerary for the order base, if a specific itinerary must be used.
ORDER_TYPE_GID
ORDER_TYPE_GID contains the unique identifier for the order type. Examples of order types include: CUSTOMER_ORDERS INTERFACILITY RETURNS INTEGRATION WEB
TIME_WINDOW_EMPHASIS_GID
TIME_WINDOW_EMPHASIS_GID contains the unique identifier to tell the planning engine whether early or late (or both) pickup and delivery times are important when creating a shipment.
RATE_SERVICE_PROFILE_GID
A unique identifier to define the rate servoce profile for the order. The rating engine uses it as a deafult if one is specified.
SERVPROV_GID
SERVPROV_GID contains the unique identifier for the carrier for the order base. The system uses it as a default, if specified.
SERVPROV_PROFILE_GID
SERVPROV_PROFILE_GID contains the unique identifier for the group of service providers that belong in the profile. The system will attempt to select a service provider from the group if a profile is specified.
TRANSPORT_MODE_GID
TRANSPORT_MODE_GID contains the unique identifier for the transportation mode for the order base. Examples of transportation modes include: AIR LTL PARCEL RAIL TL
MODE_PROFILE_GID
MODE_PROFILE_GID contains the unique identifier for the transportation mode profile that the order can use. The system tries to pick one from the group if provided here. Examples of modes include: AIR LTL RAIL TL VESSEL
EQUIPMENT_GROUP_GID
EQUIPMENT_GROUP_GID contains the unique identifier for the equipment group for the order base.
EQUIPMENT_GROUP_PROFILE_GID
EQUIPMENT_GROUP_PROFILE_GID contains the unique identifier for the equipment group profile for the order base.
PROCESSING_CODE_GID
A unique pointer to define the processing codes in the system. Currently GC3 has only one processing code "PLN" which means to plan the shipment from the order release.
SHIP_WITH_GROUP
Not used.
EFFECTIVE_DATE
Effective date of the order.
EXPIRATION_DATE
The expiration date of the order.
IS_TEMPLATE
A tru/false flag to identify if this order is just a template and order release(s) can be created out of it.
RATE_OFFERING_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
RATE_GEO_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
SELL_RATE_OFFERING_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
SELL_RATE_GEO_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
FIXED_SELL_ITINERARY_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
TOTAL_WEIGHT_BASE
TOTAL_WEIGHT in base units.
TOTAL_VOLUME_BASE
TOTAL_VOLUME in base units.
DIM_RATE_FACTOR_GID
The dim factor to be used when allocating by dimensional weight for a business unit.
USER_DEFINED1_ICON_GID
Can store a user-defined icon that will appear within the user interface to help visually identify attributes of the order base.
USER_DEFINED2_ICON_GID
Can store a user-defined icon that will appear within the user interface to help visually identify attributes of the order base.
USER_DEFINED3_ICON_GID
Can store a user-defined icon that will appear within the user interface to help visually identify attributes of the order base.
USER_DEFINED4_ICON_GID
Can store a user-defined icon that will appear within the user interface to help visually identify attributes of the order base.
USER_DEFINED5_ICON_GID
Can store a user-defined icon that will appear within the user interface to help visually identify attributes of the order base.
BUNDLING_TYPE
AUTOMATIC or MANUAL ONLY allow bundling; they are distinguished only by user defined queries. DO NOT BUNDLE means order will never be bundled with other orders.
TOTAL_DECLARED_VALUE
Sum of declared value on each OB_LINE and/or OB_SHIP_UNIT. Does not include declared value from OB_SHIP_UNIT_CONTENT.
SELL_SERVPROV_GID
indicates the sell side servprov to use for this order base
SELL_SERVPROV_PROFILE_GID
indicates the sell side servprov profile to use for this order base
RATE_SERVICE_GID
Rate service constraint for the order
PRIORITY
This column defines the priority associated with an Order Base
MARKED_FOR_PURGE
A boolean field. When it is set to "Y", this row and its child data will be purged.
