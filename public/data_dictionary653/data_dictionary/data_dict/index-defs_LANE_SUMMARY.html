Definitions - LANE_SUMMARY

Summarizes historical workload on each lane

BEGIN_DATE
When building the lane summary, only shipments whose start date is greater than begin_date are considered
END_DATE
When building the lane summary, only shipments whose start date is less than end_date are considered
SAVED_QUERY_GID
In addition to using begin_date, end_date, etc, you may specify an arbitrary saved query to indicate arbitrarily complex shipment filtering
TIME_PERIOD_SIZE
Defines the width of each time period in days. The number of time periods is than determined by the begin_date and end_date. Example: begin_date = 1/1/2005, end_date=12/31/2005. Time_period_size = 7. In this case, there would be 52 time periods
BEST_SRC_GEO_HIERARCHY_GID
Defines the best source geography - this is used when its necessary to generate a lane in procurement - see requirements doc for more information
BEST_DST_GEO_HIERARCHY_GID
Defines the best dest geography - this is used when its necessary to generate a lane in procurement - see requirements doc for more information
