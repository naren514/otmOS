Definitions - Q_INTEGRATION_IN

Q_EVENT_SEQ
Queue sequence for the inbound integration.
Q_PREEMPTION_PRIORITY
A sequence used for queue preemption.
Q_START
Time the event was last queued. If the event is scheduled, this will equal the scheduled time. If resubmitted, the resubmit time. Otherwise it matches the insert date.
Q_STATE
Queue state of the event (QUEUED, ACTIVE, CANCELED, PROCESSED, FAILED)
Q_TIME
Time the event spent in queue
Q_TIME_UOM_CODE
Unit of measure for the q_time.
Q_TIME_BASE
Database base value for the q_time.
Q_PROCESS_START
Time the event was last dequeued and passed to the executor for processing.
Q_PROCESS_TIME
Time the event took to execute.
Q_PROCESS_TIME_UOM_CODE
Unit of measure for the q_process_time.
Q_PROCESS_TIME_BASE
Database base value for the q_process_time.
Q_LOG_PROCESS_ID
ID linking the event execution to the system logs.
Q_PROCESS_MACHINE_GID
The application machine that polled and executed the event
Q_APP_SERVER_GID
The application server that polled and executed the event
Q_RELATED_QUEUE_SEQ
Sequence of the parent queue event that populated this event.
P_QUEUE_START_KEY
Index for date-based partitioning.
I_TRANSMISSION_NO
Specifies the transmission number for the event.
TOPIC_DESCRIPTION
A short description of the topic, displayed on the inbound integration events finder.
TOPIC_CLASS_NAME
Workflow topic class of the event.
TOPIC_PARAMETERS
Workflow parameters for the event.
Q_RELATED_QUEUE_NAME
Queue table name of parent event
Q_QUEUE_NAME
Queue name used to enqueue this record
IS_LONG_RUNNING_TASK
If Y, this transmission is a long running task and should be monitored in the LONG_RUNNING_TASK table.
