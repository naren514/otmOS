Definitions - AUTO_ASSIGN_CRITERIA

Content provided by GC3...maps the valid criteria that may be used to configure auto-assignment. Also provides the information to the auto-assign engine to get the data from the business object.

AUTO_ASSIGN_CRITERIA_GID
Id for the criteria that GC3 can look at to decide on auto assignments
CRITERIA_TRANSLATION_GID
Translation id for the auto assign criteria
DATA_TYPE1
Data type of the field to show when the user creates a list of items for the component; may be S/B/I/N/D/G/U/C/P/H/L/F for String, Boolean, Integer, Number, Double, Gid, UOM, Char, Percentage, Geo Hierarchy, Drop-down list, Fixed Values. If null, no field should be shown.
GID_QUERY_CLASS1
Query class to use if DATA_TYPE is a finder
VALID_VALUE_TYPE1_GID
If DATA_TYPE is F, indicates the key for the set of fixed values to be displayed in a picklist
UOM_TYPE1
Uom type to use if DATA_TYPE is a uom such as DISTANCE, CURRENCY, WEIGHT etc.
DATA_TYPE2
Same as DATA_TYPE1, but may be used if there is a second field for a single piece of criteria
GID_QUERY_CLASS2
Same as GID_QUERY_CLASS1, but may be used if there is a second field for a single piece of criteria
VALID_VALUE_TYPE2_GID
Indicates the key for the set of fixed values to be displayed in a picklist
UOM_TYPE2
Same as UOM_TYPE1, but may be used if there is a second field for a single piece of criteria
DATA_FIELD1
Field in the implied business object to get the data for the component
DATA_FIELD2
Same as DATA_FIELD1 but may be used if there is a second field for a single piece of criteria
JAVA_CLASS
Java class to get the data for the component if it is not directly accessible by a field name in the implied business object
RESULT_HANDLER_CLASS
Java class to handle checking the data from the business object against the list defined in the rule
