Definitions - LEG

The leg defines the path between the source and destination. It identifies other constraints like mode of transportation (only trucks allowed, air mode etc.)

LEG_GID
LEG_GID contains the unique identifier for the leg.
LEG_XID
LEG_XID contains the unique external identifier for the leg.
SOURCE_LOCATION_GID
SOURCE_LOCATION_GID contains the unique identifier for the source location for the leg.
DEST_LOCATION_PROFILE_GID
DEST_LOCATION_PROFILE_GID contains the unique identifier for the destination location profile for the leg.
DEST_LOCATION_RADIUS
DEST_LOCATION_RADIUS contains the radius for the destination location for the leg.
DEST_LOCATION_RADIUS_UOM_CODE
Unit of measure for DEST_LOCATION_RADIUS column.
DEST_LOCATION_RADIUS_BASE
DEST_LOCATION_RADIUS in base units.
CREATE_SINGLE_SHIPMENT
Flag indicating whether or not there should be one Shipment created per Equipment needed on this leg (N) or one Shipment for all Equipment needed on this leg (Y)
SOURCE_LOCATION_ROLE_GID
SOURCE_LOCATION_ROLE_GID contains the unique identifier for the source location role for the leg.
DEST_LOCATION_GID
DEST_LOCATION_GID contains the unique identifier for the destination location for the leg.
DEST_LOCATION_ROLE_GID
DEST_LOCATION_ROLE_GID contains the unique identifier for the destination location role for the leg.
SERVPROV_PROFILE_GID
SERVPROV_PROFILE_GID contains the unique identifier for the service provider's profile for the leg.
EQUIPMENT_GROUP_PROFILE_GID
EQUIPMENT_GROUP_PROFILE_GID contains the unique identifier for the equipment group profile for the leg.
MODE_PROFILE_GID
MODE_PROFILE_GID contains the unique identifier for the mode profile for the leg.
QUALITY_FACTOR_PROFILE_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
EXPECTED_TRANSIT_TIME
EXPECTED_TRANSIT_TIME contains the expected time for this leg of the shipment to take.
EXPECTED_TRANSIT_TIME_UOM_CODE
Unit of measure for EXPECTED_TRANSIT_TIME column.
EXPECTED_TRANSIT_TIME_BASE
EXPECTED_TRANSIT_TIME in base units.
EXPECTED_COST
EXPECTED_COST contains the expected cost for the leg.
EXPECTED_COST_BASE
EXPECTED_COST in base units.
EXPECTED_COST_CURRENCY_GID
EXPECTED_COST_CURRENCY_GID contains the currency for the expected cost of the leg.
EQUIPMENT_ASSIGNMENT_TYPE
Flag indicating how equipment groups are assigned for this Leg. A value of 'N' indicates that No Equipment will be used (i.e. it's loose freight), 'O' stands for Optimize Equipment (i.e. run through Container Optimization), and 'R' stands for Re-use Equipment from Previous Leg (or Optimize if no previous equipment is available).
RATE_OFFERING_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
RATE_GEO_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
X_DOCK_LOCATION_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
HAZMAT_REGION_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
INTERMEDIARY_CORPORATION_GID
A unique identier that represents the corporation that the location represents. The gid contains the domain portion.
RULE_11_INDICATOR
If not null Identifies the Leg as being part of a rail rule_11 Itinerary segment and determines if the shipment built from the Leg is tendered or not.
RAIL_INTER_MODAL_PLAN_GID
This is the rail inter-modal plan gid. I.e.) Plan 20 - door-to-door, Plan 25 - ramp-to-ramp, etc.
CUSTOMER_RATE_CODE
This code signifies what rate is used on a Leg, either 'C' for the Customer's rate, or 'S' for the Shipper's rate.
COFC_TOFC
Container-On-Flat-Car_Trailer-On-Flat-Car code used along with the Rail_Inter_Modal_Plan_gid to determine the rate.
SERVPROV_ASSIGNMENT_TYPE
Valid values are O and R, which stand for Optimize and Re-use. "Optimize" will be the default and allow GC3 to work the way it does currently where it finds the best service provider for each leg. "Re-use" will force GC3 to keep the same service provider on this leg that was used on the previous one.
PAYMENT_METHOD_CODE_GID
Payment method on leg which will be copied to the corresponding shipment.
VIA_SRC_LOC_PROFILE_GID
List of valid source VIA locations to evaluate in rating and routing.
VIA_DEST_LOC_PROFILE_GID
List of valid destination VIA locations to evaluate in rating and routing.
VIA_SRC_LOC_GID
Used when only a single VIA source location is valid, will be used in rating and routing.
VIA_DEST_LOC_GID
Used when only a single VIA destination location is valid, will be used in rating and routing.
PLAN_TO_OPERATIONAL_LOC
If Y, shipments for this leg should be planned to a specific operational location such as a terminal at a port
TRANS_ARB_SEARCH_TYPE
Describes when to search for trans-shipment arbitraries
