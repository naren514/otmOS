Definitions - GTM_STRUCTURE_REFNUM_QUAL

This table contains the qualifier name for the reference link to the customer internal identification to an trade item structure. For e.g., the customer may have ERP or PLM applications internally to specify the structure.

GTM_STRUCTURE_REFNUM_QUAL_GID
GTM_STRUCTURE_REFNUM_QUAL_GID contains the unique identifier for the trade item structure refnum qualifer.
GTM_STRUCTURE_REFNUM_QUAL_XID
GTM_STRUCTURE_REFNUM_QUAL_XID contains the unique external identifier for the trade item structure refnum qualifer.
GTM_STRUCTURE_REFNUM_QUAL_DESC
DESCRIPTION contains text for the description of the trade item structure refnum qualifier
DEFAULT_REFNUM_BN_TYPE_GID
the BN type used to generate a refnum when the refnum field for a record in the corresponding refnum table is not populated.
UPDATE_FLAG
Specifies if the refnum qualifier can exist on the business object multiple times, or is restricted to occur only once and whether the value is allowed to be updated. If the value is MANY, then the object can have multiple values for the same qualifier. If the value is either UPDATE_OK or UPDATE_NOT_OK, then the rule is in effect, meaning only one value is allowed for a given qualifier. In the case of UPDATE_NOT_OK, the value may not be modified.
