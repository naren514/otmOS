Definitions - S_SHIP_UNIT

This table stores the shipment ship unit information. This table was added to identify the actual shipment ship unit as opposed to order release ship unit (in earlier versions of GC3).

S_SHIP_UNIT_GID
A unique pointer to define the shipment ship unit. The gid contains the domain portion.
S_SHIP_UNIT_XID
Similar to the gid except for the domain portion.
UNIT_LENGTH
The length of each s_ship unit. If this row represents 20 same ship units, total length is 20 x UNIT_LENGTH.
UNIT_LENGTH_UOM_CODE
Unit of measure for UNIT_LENGTH column.
UNIT_LENGTH_BASE
UNIT_LENGTH in base units.
UNIT_WIDTH
: it is the weight of each s_ship_unit (each row may represent SHIP_UNIT_COUNT number of same ship units). If this field is null, or is 0, the weight of this s_ship_unit is computed as the total of container weight from its ship_unit_spec and the weight of its s_ship_unit_lines mutilply.
UNIT_WIDTH_UOM_CODE
Unit of measure for UNIT_WIDTH column.
UNIT_WIDTH_BASE
UNIT_WIDTH in base units.
UNIT_HEIGHT
The height of each s_ship unit. If this row represents 20 same ship units, total height is 20 x UNIT_HEIGHT.
UNIT_HEIGHT_UOM_CODE
Unit of measure for UNIT_HEIGHT column.
UNIT_HEIGHT_BASE
UNIT_HEIGHT in base units.
UNIT_WEIGHT
The weight of the ship unit.
UNIT_WEIGHT_UOM_CODE
Unit of measure for UNIT_WEIGHT column.
UNIT_WEIGHT_BASE
UNIT_WEIGHT in base units.
UNIT_VOLUME
it is the volume of each s_ship_unit unit (each row may represent SHIP_UNIT_COUNT number of same ship units). If this field is null, or is 0, the volume of this s_ship_unit is computed as the total of container volume from its ship_unit_spec and the volume of its s_ship_unit_lines.
UNIT_VOLUME_UOM_CODE
Unit of measure for UNIT_VOLUME column.
UNIT_VOLUME_BASE
UNIT_VOLUME in base units.
FLEX_COMMODITY_CODE
The user defined commodity code for the item. For example HTS, SITC, STCC, NMFC etc.
FLEX_COMMODITY_QUAL_GID
A unique identifier to define the tyoe of teh commodity code. For example HTS, SITC, STCC, NMFC etc.
IS_SPLITABLE
A true or false value to define whether the ship unit can be splitted when creating the shipments.
SHIP_UNIT_COUNT
Ship units that have the same demension/weight and contents are represented as one row in the SHIP_UNIT table. SHIP_UNIT_COUNT is the number of these ship units.
RECEIVED_WEIGHT_BASE
RECEIVED_WEIGHT in base units.
RECEIVED_VOLUME_BASE
RECEIVED_VOLUME in base units.
CORE_DIAMETER
For cylindrical objects, this defines the diameter of the core
CORE_DIAMETER_UOM_CODE
For cylindrical objects, unit of measure for the diameter of the core
CORE_DIAMETER_BASE
For cylindrical objects, the base value of the diameter of the core. For cylindrical objects, the base value of the diameter of the core
T_SHIP_UNIT_GID
This is used as a non-foreign key reference to the order release ship unit.
IS_COUNT_SPLITTABLE
Indicates whether the ship unit can be split on the count boundaries when planning during container optimization.
MARKED_FOR_PURGE
A boolean field. When it is set to "Y", this row and its child data will be purged.
IS_ALLOW_REPACK
If this flag is set to false, when planning order movement that contains this ship unit or combine shipments that contains this s ship unit, the shipment ship unit building process will not repack this ship unit.
PRIORITY
This column defines the priority associated with ship unit
