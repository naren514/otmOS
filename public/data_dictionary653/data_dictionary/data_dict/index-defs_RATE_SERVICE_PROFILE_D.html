Definitions - RATE_SERVICE_PROFILE_D

Groups together rate services. Can be used if multiple delivery options might be OK. Example: rate_service 1 = 'Overnight' rate_service 2 = 'Two-day' We could create a profile called 'Under two days', and then associate each of the rate services with this profile within the RATE_SERVICE_PROFILE_D table. The profile links to the OB_ORDER_BASE table.
