Definitions - GL_ACCOUNT_POLICY

Holds configurable parameters for GC3 account policies. This includes when GC3 user'spasswords should expire, whether to maintain a login history for a user, etc. The policy is then associated to a user account within the GL_USER table.

GL_ACCOUNT_POLICY_GID
The unique global identifier for the account policy.
GL_ACCOUNT_POLICY_XID
The external identifier for the account policy.
PW_EXPIRATION_DURATION
Identifies a length of time (in days) before a new password will expire. A null value indicates the password never expires.
PW_EXPIRATION_WARNING_PERIOD
Identifies the length of time(in days) prior to a user'spassword expiration in which the user should receive password expiration messages.
NUM_PW_FOR_HISTORY
The number of passwords which should be kept within the history for each user. The user will not be able to set his password to any password that is kept within the history.
PW_CHANGE_AFTER_RESET
Determines whether or not a password preset by the system or reset by an account ADMIN user via the User Manager screen will expire upon the first user login.
MAX_ATTEMPTS_ACCOUNT_LOCKOUT
The maximum number of login attempts allowed before an account is locked out. If the number is zero, then there is no lockout.
LOCKOUT_DURATION
How long to lock out an account after exceeding allowed login attempts.
LOCKOUT_DURATION_UOM_CODE
The unit of measure for the lockout duration.
LOCKOUT_DURATION_BASE
The lockout duration in base units.
MAX_DAYS_ACCOUNT_DORMANCY
Allowed inactive duration (in days) before an account is disabled.
KEEP_LOGIN_HISTORY
Whether or not to maintain a login history for the users.
