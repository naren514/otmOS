Definitions - SHIPMENT

Table (and associated tables of SHIPMENT_STOP, SHIPMENT_STOP_D, S_SHIP_UNIT_LINE, and S_SHIP_UNIT) was added in 2.5 to indicate what was actually shipped. This module was added because in 2.1 if the quantity received was different than what was requested, the ORDER_RELEASE information got overwritten. There was no way of comparing how accurate the actual shipments were to what was requested. These tables look very similar to the tables associated with ORDER_RELEASE. The key to this table is SHIPMENT_GID, because there is one record for every actual shipment in this table.
