Definitions - SHIPMENT_STOP

This table stores departure and arrival information about every stop along the way for the life of the shipment (both planned and actual). The key is both the SHIPMENT_GID and the STOP_NUM.

SHIPMENT_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
STOP_NUM
This field defines the stop number within the shipment. Stop Num = 1 is the Pickup stop, and the max (Stop Num) for a shipment is the delivery stop.
LOCATION_GID
A unique identifier to define the locations withing GC3. The gid contains the domain portion.
LOCATION_ROLE_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
PLANNED_ARRIVAL
The planned date and time of arrival at the stop.
ESTIMATED_ARRIVAL
The estimated time of arrival at the stop. It represents the up to the minute ETA that takes into account ETA updates. This date is editable by the carrier or load planner and should incorporate any known delays to delivery. It represents to the best of our knowledge, when we expect the shipment to arrive.
ACTUAL_ARRIVAL
The actual arrival date and time at the stop.
PLANNED_DEPARTURE
The planned departure date and time at the stop.
ESTIMATED_DEPARTURE
The estimated time of departure for the stop. It represents the up to the minute ETA that takes into account ETA updates. This date is editable by the carrier or load planner and should incorporate any known delays to departure. It represents to the best of our knowledge, when we expect the shipment to depart from the stop.
ACTUAL_DEPARTURE
The actual departure date and time at the stop.
WAIT_TIME
The time the truck or the transportation mode has to wait at the stop.
WAIT_TIME_UOM_CODE
Unit of measure for WAIT_TIME column.
WAIT_TIME_BASE
WAIT_TIME in base units.
REST_TIME
The time the truck or the transportation mode should take for rest at the stop.
REST_TIME_UOM_CODE
Unit of measure for REST_TIME column.
REST_TIME_BASE
REST_TIME in base units.
DRIVE_TIME
The drive time to teh stop from the last stop.
DRIVE_TIME_UOM_CODE
Unit of measure for DRIVE_TIME column.
DRIVE_TIME_BASE
DRIVE_TIME in base units.
DIST_FROM_PREV_STOP
The distance from the last stop.
DIST_FROM_PREV_STOP_UOM_CODE
Unit of measure for DIST_FROM_PREV_STOP column.
DIST_FROM_PREV_STOP_BASE
DIST_FROM_PREV_STOP in base units.
ACTIVITY_TIME
The time spent at the stop for pickup or delivery.
ACTIVITY_TIME_UOM_CODE
Unit of measure for ACTIVITY_TIME column.
ACTIVITY_TIME_BASE
ACTIVITY_TIME in base units.
ACCESSORIAL_TIME
The sum of fixed_accessorial_time for a particular stop.
PARENT_LOCATION_GID
If an operational location, such as terminal at a port, is chosen for this stop, the LOCATION_GID will be copied to this field and the LOCATION_GID field will contain the operational location.
STOP_REQUIREMENT_TYPE
This field is currently used only for TRIPs.
ARBITRARY_TYPE
SOURCE or DEST to indicate the type of arbitrary. If null, this stop is not an arbitrary.
IS_FIXED_DISTANCE
When set to Y, indicates the existence of a user-entered value for the dis_from_prev_stop field
VOYAGE_GID
Refers column VOYAGE.VOYAGE_GID.
IS_MOTHER_VESSEL
Desginating a leg as mother vessel to automatically populated header.
FLIGHT_INSTANCE_ID
Refers column FLIGHT_INSTANCE. FLIGHT_INSTANCE GID.
STOP_TYPE
Define the type of this stop.
DRIVER_NON_PAYABLE
Allows the user to specify that stop is payable or not.
DISTANCE_TYPE
L - Loaded, B- Bobtail, D - Deadhead
RATE_DISTANCE_GID
This column would give the rate distance to be used when finding distance from this stop to next stop. This is done mainly for arbitrary case.
ROUTE_CODE_GID
This column would give the route code to be used when finding distance from this stop to next stop This is done mainly for handling arbitrary case.
