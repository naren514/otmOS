Definitions - ORDER_RELEASE

This table is linked to the OB_ORDER_BASE table. It specifies the release information against the base order information. An order release can have only single origin and single destination. All line items for the order release information must have the same time windows.

ORDER_RELEASE_GID
A unique identifier created by the system for the order release. the gid contains the domain portion.
ORDER_RELEASE_XID
Same as gid except for the domain portion.
ORDER_RELEASE_NAME
A name given to the order release by the user.
ORDER_RELEASE_TYPE_GID
A unique identifier to define the type of the order release. For example: Customer order Distribution order Manufacturing order Purchase order etc.
ORDER_BASE_GID
A unique identifier to define the order base in the system. It is system generated. The gid contains the domian portion.
SOURCE_LOCATION_GID
A unique identifier to define the locations withing GC3. The gid contains the domain portion.
DEST_LOCATION_GID
A unique identifier to define the locations withing GC3. The gid contains the domain portion.
FIXED_ITINERARY_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
ASSIGNED_ITINERARY_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
FIXED_SELL_ITINERARY_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
SHIP_UNIT_ORIGIN
Indicates how the ship units were created (BL=order base by line, BS=order base by ship unit, RL=order release by line, RS=order release by ship unit). If this field is not 'RS' (formerly 'U'), GC3 deletes all existing order ship unit for the given order release, before calling new releasing logic to generate ship unit.
TOTAL_WEIGHT_BASE
TOTAL_WEIGHT in base units.
TOTAL_VOLUME_BASE
TOTAL_VOLUME in base units.
TOTAL_ITEM_PACKAGE_COUNT
Summary of ITEM_PACKAGE_COUNT of all line items.
MAX_LENGTH_BASE
MAX_LENGTH in base units.
MAX_WIDTH_BASE
MAX_WIDTH in base units.
MAX_HEIGHT_BASE
MAX_HEIGHT in base units.
PLAN_FROM_LOCATION_GID
A unique identifier to define the locations withing GC3. The gid contains the domain portion.
PLAN_TO_LOCATION_GID
Must be Y or N. Y indicates that the order should not be bundled with any others. Order releasing logic will copy this value from the order base when the order is released.
BULK_PLAN_GID
If the order was planned via the bulk plan process, then the global identifier for the bulk plan.
SELL_BULK_PLAN_GID
The bulk plan gid for a sell side bulk plan for this order.
TOTAL_SHIP_UNIT_COUNT
Number of ship units on this order.
LOAD_POINT
source load point
UNLOAD_POINT

PICKUP_IS_APPT

DELIVERY_IS_APPT

PLAN_FROM_LOAD_POINT

PLAN_TO_UNLOAD_POINT

BUNDLING_TYPE
