Definitions - BULK_CM

Table that provides information about what shipments were selected and what shipments were linked during a given run of bulk continous moves.

BULK_CM_GID
Unique identifier for the bulk continuous move process.
BULK_CM_XID
External identifier for the bulk continuous move process.
QUERY_NAME
Bulk continuous move input parameter.
START_TIME
Start time of the bulk continuous move.
END_TIME
End time of the bulk continuous move.
NUM_OF_SHIPMENTS_SELECTED
Number of shipments selected for the bulk continuous move.
NUM_OF_SHIPMENTS_LINKED
Number of shipments successfully linked by the bulk continuous move.
LOG_PROCESS_ID
The log id for the bulk continuous move process.
APP_MACHINE_GID
Application machine id the process is running on, in order to get the log from the correct machine.
BULK_PLAN_GID
If CM was initiated from a bulkplan, the bulk plan ID
NUM_SHIPMENTS_CONSIDERED
Some selected shipments might be excluded because they are on CM. This number shows the shipments that actually go into the algorithm.
NUM_OF_CM_CREATED
Number of continuous moves created though the bulk cm process.
TOTAL_EMPTY_DISTANCE
Sum of All Empty Distances in a CM added over all CMs.
CM_WITH_TWO_SHIPMENTS
Number of continous move creates with 2 shipments each.
CM_WITH_THREE_SHIPMENTS
Number of continous move creates with 3 shipments each.
CM_WITH_FOUR_SHIPMENTS
Number of continous move creates with 4 shipments each.
CM_WITH_FIVE_SHIPMENTS
Number of continous move creates with 5 shipments each.
CM_WITH_SIX_OR_MORE_SHIPMENTS
Number of continous move creates with 6 or more shipments each.
STATE
Execution state of the bulk continous move. SUBMITTED/RUNNING/COMPLETED.
DESCRIPTION
A description of the bulkcm used to identify a set of performance or quality conditions. Used to summarize the bulkcm assumptions when comparing multiple plans.
