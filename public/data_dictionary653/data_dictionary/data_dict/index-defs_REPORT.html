Definitions - REPORT

The REPORT table contains the list of standard and customized reports. One record per report.

REPORT_GID
GC3 Global indetifier for the entity. Is created by concatenating Domain Name,"." and XID.
REPORT_DESC
Report description.
REPORT_GROUP_GID
GC3 Global indetifier for the entity. Is created by concatenating Domain Name,"." and XID.
DOMAIN_NAME
Domain of the User. Is used for creating the GID.
REPORT_XID
GC3 Unique indetifier for the entity. Is used for creating the GID
REPORT_DISPLAY_NAME
The name of the report that will appear in the GC3 application.
REPORT_PHYSICAL_NAME
The actual file name of the report.
ADDITIONAL_URL_ARGUMENTS
Contains any additional URL arguments that should be called when the report is called within the GC3 application.
CAN_AUTO_GENERATE
Specifies if the report can be included in a report set. Generally only the reports that take a shipment_gid or order_base_gid as its parameter may only be set for auto generation.
SECURITY_LEVEL
If a value is present, it specifies the security level when sending the report. If no value is present, the default is taken from the properties file.
IS_RPT_MGR_DISPLAY
Flag that defines whether this report will appear on the Report Manager screen or not. Valid values are Y and N.
REPORT_FROM_DB
Default database name on which the report should be run.
SEED_DATA
Specifies if the Report is a seed data.
INSERT_USER
For audit trail purposes. This field stores the id of the user who first entered the record.
INSERT_DATE
For audit trail purposes. This field stores the date and time when the record was first entered in the database.
UPDATE_USER
For audit trail purposes. This field stores the id of the user who last updated the record.
UPDATE_DATE
For audit trail purposes. This field stores the date and time when the record was updated in the database.
USE_PARAMS_AS_BIND
Flag to determine whether to build the report parameters into valid SQL value(s) to be used as bind values or to use the finder string value(s).
THIRD_PARTY_CONTENT_TYPE
Type of support for 3rd party report generation
USE_THIRD_PARTY_DISTRIBUTION
Flag to control whether a 3rd party is responsible for report distribution
REPORT_SYSTEM_GID
Report System generating the report. If left blank, embedded BI Publisher is used.
REPORT_PATH
The path of the report. Report systems use this path, along with a path prefix to determine which report to generate.
USE_PARAM_OPERATORS
Flag specifying whether finder operators should be used for report parameters. Note that this is independent of use_params_as_bind. That flag determines whether the report parses whatever arguments it receives and uses bind variables for queries. You can have a report that uses bind variables and finder operators, or one that uses concatenation without operators.
DATA_QUERY_TYPE_GID
Data query type associated with the report. Used for optional document storage.
