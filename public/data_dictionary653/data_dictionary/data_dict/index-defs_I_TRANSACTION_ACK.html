Definitions - I_TRANSACTION_ACK

The table contains the Transaction Ack that are received for outbound Transmissions.

I_TRANSACTION_NO
The Transaction numer referring to the Transaction for the Ack.
SEQUENCE_NO
Sequence number to uniquely identify the Transaction Ack for the Transaction.
ACK_STATUS
Specifies the status provided in the Transaction Ack.
ACK_REASON
Specifies the reason for the status provided within the Transaction Ack.
REF_TRANSACTION_ID
Represents a Transaction ID assigned for the Transaction from the external system.
REF_TRANSACTION_NO
Represents the Transaction record containing the xml for this Transaction Ack.
PARTITION_KEY
Populated through a trigger, this key is used to determine into which partition this record should be inserted. The formula is based upon the insert_date column and the frequency in which this table is associated (weekly, monthly, quarterly). Valid values are 1-4.
