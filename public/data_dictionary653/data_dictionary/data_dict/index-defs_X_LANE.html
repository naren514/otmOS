Definitions - X_LANE

This table defines the lanes between any two points. The two points can be: - predefined locations (from the Location table), - predefined regions (from the Region and Region Detail tables), - Hierarchical locations (from Geo_hierarchy table), or - a combination of any part of the address that includes City, Province, Postal Code, Country or Zone(s). The zones can be any user defined entries including SPLC (Standard Point Location Codes)

X_LANE_GID
A unique identifier to define the X lanes in the GC3. The gid contains the domain portion. The X-Lane is the user interface in GC3 to define all the lanes.
X_LANE_XID
GC3 Unique identifier for the entity. Is used for creating the GID
SOURCE_LOCATION_GID
A unique identifier to define the locations withing GC3. The gid contains the domain portion.
SOURCE_CITY
The city of the source portion of the location.
SOURCE_PROVINCE_CODE
The state code for teh source portion of the X-Lane.
SOURCE_POSTAL_CODE
The zip code of the source location portion of the X-Lane.
SOURCE_COUNTRY_CODE3_GID
A pointer to the three digit country code for teh source portion of the X-Lane.
SOURCE_ZONE4
The Zones for the source location portion of the X-Lane.
SOURCE_ZONE1
The Zones for the source location portion of the X-Lane.
SOURCE_ZONE2
The Zones for the source location portion of the X-Lane.
SOURCE_ZONE3
The Zones for the source location portion of the X-Lane.
SOURCE_GEO_HIERARCHY_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.

SOURCE_REGION_GID
A unique identifier to define the region code for the source portion of the X-Lane. This is a way to define the X-Lane from a region to another region (or any combination)
