Definitions - SPECIAL_SERVICE

This is the master table that holds various special services required for transportation, pickup and delivery. A special service differs from an accessory in that the customer does not have to pay for using the special service. An example of a special service can be a pump required to pull the chemicals from a delivery truck. The special services defined can be linked at the item level or at the location level in GC3. The logic in GC3 checks to see if a special service is required and thus, schedules the pick-up or delivery accordingly.

SPECIAL_SERVICE_GID
SPECIAL_SERVICE_GID contains the unique identifier for the special service for the item being shipped.
SPECIAL_SERVICE_XID
SPECIAL_SERVICE_XID contains the unique exterbak identifier for the special service for the item being shipped.
PAYABLE_INDICATOR_GID
indicates the default payable status of this special service
BILLABLE_INDICATOR_GID
indicates the default billable status of this special service
SPECIAL_SERVICE_DESC
SPECIAL_SERVICE_DESC contains the description for the special service for the item being shipped.
MODE_PROFILE_GID
MODE_PROFILE_GID contains the unique identifier for the mode profile for the item being shipped.
IS_ARBITRARY
Designates whether or not a special service is an arbitrary type special service.
SPECL_SRV_GROUP_GID
Allow user to organize special services by group
IS_FOR_DRIVER_CHK
Designates this special service is to be checked during driver assignment.
IS_FOR_PU_CHK
Designates this special service is to be checked during power unit assignment.
IS_FOR_EQUIP_CHK
Designates this special service is to be checked during equipment assignment.
DRIVER_LEVEL
Designates this special service is to apply at the driver or driver type level.
EQUIP_LEVEL
Designates this special service is to apply at the equipment or equipment type level.
PU_LEVEL
Designates this special service is to apply at the power unit or power unit level.
IS_FOR_RATING
Designates this special service is to be used for rating.
ACTIVITY_TYPE_GID
Hard coded list of stop activities that have internal OTM logic tied to it. This field is the one that maps the hard coded logic to the user definable notion of special service.
FOR_SOURCE_DEST
The default stop which this special service will be on
SEQ_NUM
Define this special service happens 1-before loading, 2-between unloading and loading, 3-after loading
DRIVER_TIME_TYPE
Indicates if the time associated with this special service is considered work time, drive time, rest time, or wait time for a driver, or does not count as driver time (when the value is null).
