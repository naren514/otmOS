Definitions - PROCESS_CONTROL_HISTORY

Contains an entry for each iteration of a workflow run that was kicked off from a process control request. If a process is run only once, then there is one record in the process_control_request table and the PROCESS_CONTROL_HISTORY table. However, if a process is set up to run 5 cycles, then the instructions would show up once in the PROCESS_CONTROL_REQUEST table, but 5 records would exist in the PROCESS_CONTROL_HISTORY table.
