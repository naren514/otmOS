Definitions - MONITOR_AGENT_EVENT

The MONITOR_AGENT_EVENT is a link for each agent to the list of events which were of interest to the agent and evaluated its saves queries to true.

MONITOR_AGENT_EVENT_SEQNO
The primary key for uniqueness.
MONITOR_AGENT_GID
A foreign key reference to the Monitor_Agent to which the event was of interest and evaluated its saved queries to true.
I_TRANSACTION_NO
The foreign key reference into the ie_shipment_status table for events which are xml based.
EVENT_GROUP_GID
The foreign key reference into the agent_event_group table.
EVENT_OCCUR_DATE
The date and time the event occurred.
EVENT_RECEIVED_DATE
The date and time the event was received.
EXCEPTION_MONITOR_AGENT_GID
The milestone that got an error because of the profiling exception milestone.
TRIG_EXPT_MONITOR_AGENT_GID
The milestone which caused the profiling exception determined by the exception milestone
EXCEPTION_AGENT_EVENT_GID
This is the GID of the exception event that was detected. (e.g. MONITORED SHIPMENT - MISSING This is the GID of the exception event that was detected. (e.g. MONITORED SHIPMENT - MISSING SEQUENCE MILESTONE)
IS_CURRENT_EXCEPTION
Indicates that the agent event in case of an exception event is currently valid. This value is reset to 'N' every time an agent that produced the exception event is profiled. If the exception still persists another event record is inserted with a value of 'Y'.
PREDECESSOR_MONITOR_AGENT_GID
GID of the predecessor milestone used to determine this exception.
SUCCESSOR_MONITOR_AGENT_GID
GID of the successor milestone used to determine this exception.
STATUS_CODE_GID
Status code for this event
REASON_CODE_GID
Reason code for this event
