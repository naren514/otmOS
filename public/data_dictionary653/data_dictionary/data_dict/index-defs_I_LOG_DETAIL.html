Definitions - I_LOG_DETAIL

This table is related to the I_LOG table for storing the details of the audit trail information for the XML transmissions and transactions. The I_LOG_DETAIL table stores qualifier/value pairs associated with a given log message.

I_LOG_DETAIL_SEQNO
I_LOG_DETAIL_SEQNO provides a sequence number for a transaction.
I_LOG_SEQNO
A pointer back to the log sequence number from I_Log table.
I_LOG_DETAIL_QUAL
I_LOG_DETAIL_QUAL provides the application name, transaction code and transaction number for an XML transaction.
I_LOG_DETAIL_VALUE
The actual value for log detail.
PARTITION_KEY
Populated through a trigger, this key is used to determine into which partition this record should be inserted. The formula is based upon the insert_date column and the frequency in which this table is associated (weekly, monthly, quarterly). Valid values are 1-4.
