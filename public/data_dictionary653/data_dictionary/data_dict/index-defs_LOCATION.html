Definitions - LOCATION

A Location is a place where transportation related activities occur, such as loading and unloading freight. In addition, a location may represent a corporation, and/or a service provider.

LOCATION_GID
LOCATION_GID contains the unique identifier for the location.
LOCATION_XID
LOCATION_XID contains the unique external identifier for the location.
LOCATION_NAME
LOCATION_NAME contains the name of the location.
CITY
CITY contains the name of the city for the location.
PROVINCE
PROVINCE contains the province (or U.S. state) for the location.
PROVINCE_CODE
PROVINCE_CODE contains the abbreviation for the provice (or U.S. state) for the location.
POSTAL_CODE
POSTAL_CODE contains the postal code for the location.

IS_TEMPORARY
A flag to identify if the location is created temporarily for a certain shipment or order. This location will not be used for other shipments or processes.

IS_ADDRESS_VALID

RATE_CLASSIFICATION_GID contains the unique identifier for the rate classification. Examples include: COMMERCIAL and RESIDENTIAL.

The smallest appointment slot interval for the location.
NUM_OF_ROWS_IN_YARD
The total number of rows in the yard of the location.

The number of slots per row in the yard of the location.
USE_APPOINTMENT_PRIORITY
Whether or not to consider appointment priority
SCHEDULE_LOW_PRIORITY_APPOINT
Whether or not to schedule appointment for low priority appointment that has been bumped off.
ENFORCE_TIME_WINDOW_APPOINT
Whether or not to enforce the pick up or delivery time window during appointment schedule process."USE_APPOINTMENT_PRIORITY" takes precedence of "ENFORCE_TIME_WINDOW_APPOINT".
SCHEDULE_INFEASIBLE_APPOINT
Whether or not to schedule appointment if the appointment time available is out of pick up or delivery time window.
BB_MAX_CASES_PER_BATCH
Maximum number of cases allowed in each batch. If blank, OTM will assume that there is no limit. If the number of cases in the order exceeds this maximum, then the batch balancer's splitting logic will distribute the cases into multiple batches uniformly while honoring certain rules.
BB_IS_NEW_STORE
Indicates if this location should be treated as a new store by the Batch Balancer logic. New stores can only be assigned by Batch Balancer to a DOOR whose LOAD_NEW_STORE field is set to Y.
BB_ORDER_SPLIT_METHOD
Value of "CONVEYABLE" causes only conveyable cases to be counted toward max cases, value of "ALL" causes all cases (including non-con and repack) to count toward max cases.
EXCLUDE_FROM_ROUTE_EXECUTION
When = Y, this location is excluded from any Route Instance Legs.
