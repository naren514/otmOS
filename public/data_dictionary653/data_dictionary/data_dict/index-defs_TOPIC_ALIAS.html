Definitions - TOPIC_ALIAS

Contains a user-friendly alias for a java class associated with a workflow. Was mainly created so that a workflow could be kicked off by inserting a SQL statement into the PROCESS_CONTROL_REQUEST table through SQL*Plus, rather than having to insert it into GC3 (for testing purposes). Someone kicking it off through SQL*Plus would just need to know the alias, rather than having to type in the whole Java class in the TOPIC_CLASS_NAME column. A workflow will be kicked off whether the TOPIC_CLASS_NAME or the TOPIC_ALIAS_GID is populated within the PROCESS_CONTROL_REQUEST table.

TOPIC_ALIAS_GID
GC3 Global identifier for the entity. Is created by concatenating Domain Name,"." and XID.
TOPIC_ALIAS_XID
GC3 Unique identifier for the entity. Is used for creating the GID
IS_FOR_PROCESS_CONTROL
Specifies if the topic alias is available for use as a process control request.
PROCESS_CONTROL_MENU
Specifies the hierarchy for the process control menu.
IS_PROCESS_SYNC
Specifies whether the process is executed or published. Topics which take more than 30 seconds or so should be published to avoid browser timeout. Others should be executed to allow for immediate error reporting.
PROCESS_CONTROL_HELP
Specifies the online help file for the process topic.
IS_FOR_BATCH_PROCESS
Specifies if the topic alias is available for use in batch
IS_LONG_RUNNING_TASK
If Y, this topic is a long running task and should be monitored in the LONG_RUNNING_TASK table.
LRT_INTERRUPTABILITY
Interruptability support: NONE = task cannot be interrupted; INTERRUPTIBLE = task can be interrupted, likely ending in a failure state; RESUBMITTABLE: task can be resubmitted into it's origin queue for handling by another machine
LRT_SCA_REQUIREMENTS
For long running tasks, determines if the task should be pulled out of the PROCESS_CONTROL_REQUEST table only when resources are available to process it. NONE=task is pulled regardless of resources; THREAD=task is pulled only if an event thread in the queue is available; MEDIATOR=task is pulled only if an event thread is available and the task will not be blocked by the mediator.
LRT_INTERRUPT_ON_SQL
If 'Y', a task interruption will force the next SQL statement to fail.
