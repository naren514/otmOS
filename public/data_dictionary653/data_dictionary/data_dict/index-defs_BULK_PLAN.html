Definitions - BULK_PLAN

Table that provides information about what orders were planned and what shipments were created during a given run of bulk plan.

BULK_PLAN_GID
Global identifier generated for each call to the bulk plan process.
BULK_PLAN_XID
External identifier that is generated for each call to the bulk plan process.
QUERY_NAME
Bulk plan input parameter.
START_TIME
Start time of the bulk plan.
END_TIME
End time of the bulk plan.
NUM_OF_ORDERS_SELECTED
Number of orders selected from the database by the bulk plan.
NUM_OF_ORDERS_UNASSIGNED
Number of orders which was planned already but anassigned by the bulk plan.
NUM_OF_ORDERS_PLANNED
Number of orders successfully planned by the bulk plan.
NUM_OF_SHIPMENTS_BUILT
Number of shipments generated by the bulk plan.
PERSPECTIVE
Must be B, S, or null.
STATE
Execution state of the bulkplan. INTERRUPTED when a user has requested termination but the bulkplan is still running. TERMINATED when the bulkplan completes termination.
SINCE
Time of last state change
DESCRIPTION
A description of the bulkplan used to identify a set of performance or quality conditions. Used to summarize the bulkplan assumptions when comparing multiple plans.
SCENARIO_GROUP
A unique ID to identify a set of bulkplans for comparison
SHIPMENTS_SUPPRESSED
If Y, shipments were not commited to the database. This is a what-if scenario.
TERMINATION_TIME
Time when the bulkplan was terminated or timed out
TERMINATION_USER
User who terminated the bulkplan
TERMINATION_REASON
Reason the bulkplan was terminated
TOTAL_SHIPMENT_TIME_SECS
Sum of all shipments times (end-start) in seconds
TOTAL_WEIGHT_CAPACITY
Sum of all shipment weight capacities (based on equipment used)
TOTAL_VOLUME_CAPACITY
Sum of all shipment volume capacities (based on equipment used)
NUM_OF_ORDERS_THRU_POOL
Number of orders that went through a pool point
NUM_OF_ORDERS_THRU_XDOCK
Number of orders that went through a crossdock
TOTAL_BEST_DIRECT_COST
Total best direct cost of planned orders
NUM_OF_WA_CREATED
Number of Work assignments created during the Bulk plan process.
