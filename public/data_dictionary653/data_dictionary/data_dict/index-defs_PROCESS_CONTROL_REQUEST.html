Definitions - PROCESS_CONTROL_REQUEST

Contains an entry for each process request. An entry could indicate a one-time kickoff of a workflow, or could be an ongoing job, with a specified interval. For example, a process could be Print Invoices (topic_alias_gid), which is run every day at 6am (next_process_time). If the number_of_cycles equals "-1", then the process will resubmit itself indefinitely for another run. If the TOPIC_ALIAS_GID is null, then GC3 will look at the TOPIC_CLASS_NAME, the java class, to know what workflow to run.
