Definitions - REPORT_LOG

The REPORT_LOG table contains log information about when a report was run, and the parameters that the report was run with.

FILE_NAME
Filename generated by the report program. For forms that need to be stored for each run a new filename will be generated. For operational reports where only the current run pdf file is needed, an update will be done with the same file name.
REPORT_GID
GC3 Global indetifier for the entity. Is created by concatenating Domain Name,"." and XID.
JOB_NUMBER
Running sequence generated by the BN number generator
GL_USER_GID
GC3 Global indetifier for the entity. Is created by concatenating Domain Name,"." and XID.
RUN_DATE
The date/time when the report was run
DOMAIN_NAME
Domain of the User. Is used for creating the GID.
REPORT_SET_GID
The report set that this report was run for (this can be null).
INSERT_USER
For audit trail purposes. This field stores the id of the user who first entered the record.
INSERT_DATE
For audit trail purposes. This field stores the date and time when the record was first entered in the database.
UPDATE_USER
For audit trail purposes. This field stores the id of the user who last updated the record.
UPDATE_DATE
For audit trail purposes. This field stores the date and time when the record was updated in the database.
BULK_REPORTING_GID
If the report is part of a bulk report generation, the ID for monitoring
