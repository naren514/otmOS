Definitions - SHIP_UNIT

This table stores the unit level information at the order release level. It is a parallel table to S_SHIP_UNIT table (at the Shipment level).

SHIP_UNIT_GID
A unique identifier to define the shipping unit that will be used for the order release line. The gid contains the domain portion.
SHIP_UNIT_XID
GC3 Unique identifier for the entity. Is used for creating the GID
IS_SPLITABLE
A true or false flag to identify if the ship unit can be split when creating an order.
UNIT_LENGTH
The length of each ship unit. If this row represents 20 same ship units, total length is 20 x UNIT_LENGTH.
UNIT_LENGTH_UOM_CODE
Unit of measure for UNIT_LENGTH column.
UNIT_LENGTH_BASE
UNIT_LENGTH in base units.
UNIT_WIDTH
it is the weight of each ship_unit (each ro may represent SHIP_UNIT_COUNT number of ship units). If this field is null, or is 0, the weight of this ship_unit is computed as the total of container weight from its ship_unit_spec and the weight of its ship_unit_lines.
UNIT_WIDTH_UOM_CODE
Unit of measure for UNIT_WIDTH column.
UNIT_WIDTH_BASE
UNIT_WIDTH in base units.
UNIT_HEIGHT
The height of each ship unit. If this row represents 20 same ship units, total height is 20 x UNIT_HEIGHT.
UNIT_HEIGHT_UOM_CODE
Unit of measure for UNIT_HEIGHT column.
UNIT_HEIGHT_BASE
UNIT_HEIGHT in base units.
UNIT_WEIGHT
Weight of the shipping unit.
UNIT_WEIGHT_UOM_CODE
Unit of measure for UNIT_WEIGHT column.
UNIT_WEIGHT_BASE
UNIT_WEIGHT in base units.
UNIT_VOLUME
it is the volume of this ship_unit (each ro may represent SHIP_UNIT_COUNT number of ship units). If this field is null, or is 0, the volume of this ship_unit is computed as the total of container volume from its ship_unit_spec and the volume of its ship_unit_.
UNIT_VOLUME_UOM_CODE
Unit of measure for UNIT_VOLUME column.
UNIT_VOLUME_BASE
UNIT_VOLUME in base units.
FLEX_COMMODITY_CODE
The user defined commodity code for the item. Eg. the HTS, NMFC, SITC, STCC codes for the item.
FLEX_COMMODITY_QUAL_GID
A unique identifier to define the type of the code. Eg. HTS, SITC, STCC, NMFC etc.
IS_SYSTEM_GENERATED
A tru or false identifier to define if the record is generated by the system for the order base or copied (may be from XML).
ORDER_RELEASE_GID
A unique identifier created by the system for the order release. the gid contains the domain portion.
OR_EQUIPMENT_SEQ
Associates equipment of the order release to this particular ship unit.
SHIP_UNIT_COUNT
: Ship units that have the same demension/weight and contents are represented as one row in the SHIP_UNIT table. SHIP_UNIT_COUNT is the number of these ship units.
CORE_DIAMETER
For cylindrical objects, this defines the diameter of the core
CORE_DIAMETER_UOM_CODE
For cylindrical objects, unit of measure for the diameter of the core
CORE_DIAMETER_BASE
For cylindrical objects, the base value of the diameter of the core
IS_COUNT_SPLITTABLE
Indicates whether the ship unit can be split on the count boundaries when planning during container optimization.
PICKUP_OR_STOP_GID
This ship unit is going to be picked up at this stop. If this field is empty, this ship unit is picked up at the order's source location.
DROPOFF_OR_STOP_GID
This ship unit is going to be dropped off at this stop. If this field is empty, this ship unit is dropped off at the order's destination location.
IS_ALLOW_REPACK
If this flag is set to false, when planning order movement that contains this s ship unit or combine shipments that contains this s ship unit, the shipment ship unit building process will not repack this s ship unit.
IS_EXCLUSIVE_USE
Used to specify if the ship unit is EXCLUSIVE USE. Ship unit is not mixed with anything even if they belong to the same MIXING family if its marked as EXCLUSIVE USE.
PRIORITY
This column defines the priority associated with a ship unit
